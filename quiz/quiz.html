  3. Explain the difference between `==` and `===`. Why is strict equality usually better?
  4. == (Loose Equality): Compares only the values of two items. If the data types are different (like a number and a text string), JavaScript automatically converts one type to match the other before comparing. For example, 5 == "5" is true.
  5. === (Strict Equality): Compares both the values and the data types without converting anything. For example, 5 === "5" is false because a number is not the same data type as a text string.
  6. Why strict equality is better:
  7. It prevents unexpected bugs. Loose equality can hiddenly change your data types behind the scenes, leading to logic errors that are very difficult to track down. Strict equality ensures your conditions are precise.
  8. Give an example (in your own words) of when you would use `&&` (AND) vs `||` (OR).
  9. && (AND) when all conditions are to be true.
  10. Example: Booking a rental car. You can only rent the vehicle if you have a valid driver's license AND you are over 25 years old. If either condition is missing, you get rejected.
  11. || (OR) when only one of the conditions needs to be true. Such as movie ticket discounts. You get a cheap ticket if you are a student OR if you are a senior citizen. You do not need to be both; meeting just one requirement unlocks the discount.
