Javascript Questions

Why is textContent safer than innerHTML?

textContent treats all inputs as raw text strings. It prevents malicious script execution from dangerous user data input.


How are HTML elements connected to JavaScript variables using DOM selection?

They connect via query methods like document.querySelector(). This targets structural markup IDs or classes and maps them to a variable reference.


What would happen to the text box with id="text" if I added this line to the code above it: text.textContent = "Hello!"

The text content inside that target element would instantly overwrite to display the word "Hello!".
